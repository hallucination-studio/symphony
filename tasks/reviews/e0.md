# E0 Review

## Scope

Reviewed the E0 diff against `tasks/spec.md`, `tasks/plan.md`, and the approved
architecture sources for Podium Desktop, Linear flow, Performer Profiles, and
the Roadmap V1 boundary.

## Code simplification

- Accepted: clear completed step deadlines so successful actions do not leave
  process-lifetime timers behind.
- Accepted: remove lock-file reads and the unused owner detail from contention
  errors. The fixed sanitized reason is simpler and does not expose arbitrary
  local file content.
- No change: retained the temporary Podium store, E2E composition factory,
  backend composition factory, and Desktop client. Each names a distinct
  ownership or process boundary required by E0; inlining them would obscure the
  production/E2E separation.
- No unrelated refactors or error-path reductions were accepted.

## Five-axis review

### Required and accepted

- Correctness/performance: `StepRunner` did not cancel deadline timers after a
  step settled. Added deterministic cleanup; runner tests now exit immediately.
- Security/readability: lock contention attached raw lock-file content to the
  thrown error. Removed that content and retained only `e2e_lock_unavailable`.
- Correctness: terminal non-retryable Linear failures reported the configured
  maximum rather than the actual attempt count. Track and test actual attempts.
- Correctness: added a Desktop-client contract test covering Profile row
  reacquisition across UI rerenders, bounded secret entry, readiness, and
  activation.
- Repository hygiene: changed the ignored `tasks/` parent rule so only the
  authorized Markdown plans, ledgers, and reviews can be tracked.

### Accepted architecture and security decisions

- E2E repository selection remains feature-gated in Rust and still uses the
  production repository inspector.
- Linear client-credentials remain in one temporary Podium owner; credentials,
  SDK objects, and refresh-token substitutes do not cross into Conductor or
  browser contracts.
- WDIO receives only app-required Linear credentials and allowlisted project
  and repository inputs. Operator Linear, OpenAI, and GitHub secrets remain in
  the runner process.
- Production builds exclude WDIO plugins, WDIO capabilities, the E2E repository
  environment seam, and the E2E backend entrypoint.

### Deferred

- Real macOS packaged smoke: preflight failed closed because required local
  user Linear, OpenAI, GitHub, project-name, repository, and branch settings
  were absent. No external mutation was attempted.
- Production packaging: the release binary and `Symphony.app` bundle built,
  but Tauri's generated macOS `bundle_dmg.sh` failed while creating the DMG.
  The final DMG artifact remains unverified on this host.
- Dependency advisories: the official npm audit reports vulnerabilities through
  existing Vite/Vitest and new WDIO development-only paths, with no available
  fix for the reported chains. Toolchain replacement is not authorized by E0.

### Rejected as scope expansion

- Windows UI, ChatGPT live-login E2E, V2 multi-Root scheduling, a second
  Provider, nightly runs, automatic merge, compatibility shims, and dependency
  toolchain migration. None is authorized for E0 by `docs/architecture/` or the
  E0 scope ledger.

## Residual risk

The runner, production negative controls, feature-gated build, and local
contracts are verified. The final macOS DMG and the real
WDIO-to-Tauri-to-Linear/Codex boundary remain unverified until packaging works
on a capable host and complete allowlisted configuration is supplied for the
opt-in smoke.
