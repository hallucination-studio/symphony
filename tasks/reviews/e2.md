# E2 Review

## Scope

Reviewed E2 against S2/S3 in `tasks/spec.md`, the E2 ledger, and architecture
sources for Linear authority, Conductor restart, Performer identity, and Git
delivery convergence.

## Code simplification

- Accepted: S2 and S3 use fixed step tables with named predicates and one small
  shared driver precheck. This keeps order visible without duplicating runner
  mechanics.
- Accepted: scenario-specific test selection moved behind one small runner so
  `--scenario S2` and `--scenario S3` execute only their intended contracts.
- Retained separate S2 probe and S3 recovery evidence checks because their
  invariants differ materially.
- Rejected a generic probe/fault framework. Architecture authorizes only the
  named V1 boundaries, so a general chaos abstraction would expand scope.

## Five-axis review

### Required and accepted

- Correctness: prevalidate the complete scenario driver before any step can
  mutate external state.
- Correctness: enforce eight distinct run-owned S2 Root IDs and stop before the
  next probe after any failed expectation.
- Correctness: every S2 probe requires differing before/after remote versions,
  a Turn ID, expected state, and a closed `[E2E Probe] <id>` marker.
- Security: reject arbitrary text after the probe marker so diagnostics cannot
  masquerade as business instructions.
- Correctness: Project Label evidence requires next-Turn resolution, stale
  Result rejection, and original-Project resumption.
- Correctness: S3 Work convergence separately proves interruptions after Work
  commit, input hash, and Linear state, with no duplicate commits or markers.
- Security: Performer continuity evidence exposes only a hashed identity.
- Architecture: every S3 fault must be test-owned and surface immediately;
  Conductor recovery explicitly proves no database use.
- Correctness/readability: `acceptance:v1` now exposes exact S2/S3 dry-run plans
  and scenario-specific opt-in guards without producing passed verdicts.

### Deferred by explicit user authorization

- Concrete probe/fault fixtures and real S2/S3 execution against Linear, Codex,
  Git/GitHub, and packaged Desktop. Missing configuration remains fail-closed
  and the live verdict remains incomplete.

### Rejected as scope expansion

- External-service fault injection, a general chaos platform, checkpoint or
  journal state, Conductor persistence, mirrored workflow queues, S1 changes,
  CI workflow changes, Windows UI, V2 scheduling, automatic merge, and
  compatibility paths.

## Residual risk

The ordering, isolation, evidence shape, and stop conditions are contract-tested
only. Recovery and authority claims are not proven against real process and
external-system boundaries until the deferred live fixtures are configured and
S2/S3 are run.
