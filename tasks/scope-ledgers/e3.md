# E3 Scope Ledger

## authorized

- Add the manual Linux GitHub Actions workflow, protected Environment binding,
  global concurrency, Xvfb setup, local/CI environment mapping, and the shared
  E2E command sequence defined by `tasks/plan.md`.
- Add preflight and evidence/verdict entrypoints shared by local macOS/Linux and
  CI, with sanitized always-run artifact collection.
- Keep the automated API Key verdict separate from the incomplete full Roadmap
  verdict while ChatGPT live login is untested.

## required_consequences

- Only `workflow_dispatch` can start the real workflow; the configured GitHub
  Environment supplies review protection and dedicated E2E secrets.
- One workflow run mutates HELL at a time. S1, S2, and S3 execute strictly in
  that order after install, build, and read-only preflight.
- Missing secrets, allowlist drift, build failure, or unavailable live fixtures
  fail before mutation and cannot produce a passed verdict.
- Failure still collects sanitized, run-owned evidence and performs bounded
  run-owned cleanup without deleting another run's state.
- CI and local execution use the same `e2e:doctor`, `e2e:build`,
  `acceptance:v1`, and acceptance evaluation entrypoints.

## out_of_scope

- Automatic scheduled/push E2E runs, macOS CI matrix, Windows UI, nightly or
  soak tests, ChatGPT live login, V2 scheduling, deployment, automatic merge,
  or changes to protected Environment/branch settings through repository code.

## assumptions_requiring_approval

- None. The user authorized implementing all E2E tasks before configuring the
  concrete live fixtures and attempting the reviewed real workflow run.

## deferred_ideas

- GitHub Environment creation/reviewer assignment, concrete live fixture
  values, successful real Linux workflow evidence, ChatGPT evidence, and macOS
  CI remain external or deferred until implementation is complete.
