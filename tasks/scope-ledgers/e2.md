# E2 Scope Ledger

## authorized

- Implement the eight ordered S2 Linear-authority probes and four ordered S3
  recovery/convergence steps defined in `tasks/spec.md`.
- Add fixed coordinator contracts, dry-run plans, opt-in guards, sanitized
  evidence requirements, and fail-fast negative controls.
- Inject faults only at test-owned Performer, Conductor, Git/Linear application,
  and `gh` process boundaries already named by the architecture.

## required_consequences

- Every S2 probe uses a distinct run-owned Root and records before/after remote
  versions, Turn ID, expected state, and an explicit `[E2E Probe]` marker.
- A failed probe stops S2 immediately; no later Turn, Git operation, or Linear
  business mutation may begin.
- Project Label changes apply only at a Turn boundary and stale Results never
  advance Linear state.
- S3 resumes from current Linear, Git, and Profile facts without a Conductor
  database, checkpoint, journal, mirrored queue, or local workflow state.
- Recovery reuses stable performer identity, commits, markers, branches, and
  worktrees instead of duplicating them.

## out_of_scope

- S1 changes, CI workflow implementation, Windows UI, ChatGPT login, V2
  multi-Root scheduling, a second Provider, automatic merge, compatibility
  paths, or new product persistence/contracts.
- Fault injection into external Linear, GitHub, or Provider infrastructure.

## assumptions_requiring_approval

- None. The user authorized implementing all E2E tasks before supplying fixed
  live fixtures and attempting real external runs. Missing live fixtures must
  fail closed and cannot produce a passed verdict.

## deferred_ideas

- Concrete live probe inputs, repository fault fixtures, successful real S2/S3
  runs, Linux CI wiring, and artifact publication are deferred until all E2E
  implementation tasks are complete.
