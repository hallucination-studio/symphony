# E0 Scope Ledger

## authorized

- Roadmap V1 fixed-scenario E2E infrastructure described by `tasks/spec.md` and `tasks/plan.md` for E0.
- Configuration/preflight, fixed step execution, shared business-action contracts, Linux/macOS UI adapters, the existing repository picker/API-key secret-frame seams, and bounded Linear mutation handling.
- Automated API-key verdict reporting separately from the full Roadmap V1 verdict.

## required_consequences

- Preflight fails before any external mutation when required configuration, allowlists, runtime, or repository checks are invalid.
- Scenario steps have bounded deadlines, expected observations, evidence, and fail-fast behavior.
- Linux and macOS implement the same UI contract; platform branching stays in the UI adapter boundary.
- E2E-only seams are opt-in and production-safe by default; secrets never enter evidence, logs, browser responses, or custom persistence.
- Linear mutation retries are bounded, read back outcomes, and surface sanitized blocked/stop results after exhaustion.

## out_of_scope

- V2 multi-Root scheduling, a second Provider, ChatGPT live-login E2E, Windows UI, a general cross-platform process/display abstraction, automatic merge, or new customer-visible authentication modes.
- Real HELL/Linear/GitHub/Codex mutation during local unit/contract verification.
- Replacing or redesigning existing Podium, Conductor, Performer, or Desktop product contracts.

## assumptions_requiring_approval

- None; the user authorized execution by continuing the approved E0 task.

## deferred_ideas

- ChatGPT live-login evidence, Windows adapter, nightly/long-duration runs, and a GitHub macOS matrix remain deferred as specified by E0.
